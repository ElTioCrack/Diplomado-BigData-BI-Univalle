PROYECTO INTEGRADOR POR COMPETENCIAS
Diagnóstico Data-Driven basado en datos reales (Open Data)
1. Descripción general
El presente proyecto tiene como finalidad que cada postgraduante desarrolle un diagnóstico
organizacional basado en datos reales, utilizando datasets provenientes de fuentes abiertas
(Open Data).
El trabajo es individual, y cada postgraduante deberá seleccionar un dataset a su conveniencia.
El proyecto integra los siguientes componentes:
• comprensión del contexto de los datos
• análisis exploratorio
• definición de KPIs estratégicos
• identificación de problemas
• propuesta de mejora basada en evidencia
2. Objetivo del proyecto
Aplicar los fundamentos de:
• analítica de datos
• gobierno de datos
• KPIs estratégicos
• toma de decisiones basada en evidencia
para interpretar un dataset real y generar conclusiones útiles para la gestión organizacional.
3. Selección del dataset
Cada postgraduante deberá seleccionar un dataset de fuentes abiertas.
Condiciones obligatorias:
• dataset real (no inventado)
• dataset con suficiente información (mínimo 3 variables relevantes)
• dataset comprensible (no excesivamente complejo)

Se debe identificar:

• significado de los datos
Ejemplo:

4.3 Identificación de datos relevantes
El postgraduante deberá seleccionar las variables más importantes para su análisis.
Ejemplo:
• ingresos del cliente
• monto del crédito
• estado del crédito
Aquí se busca que se filtre y priorice información.

El postgraduante deberá realizar un análisis básico, que puede incluir:
• promedios
• comparaciones
• tendencias

4.5 Definición de KPIs estratégicos
El postgraduante deberá definir al menos 3 KPIs, basados en el dataset.
Cada KPI debe incluir:
• nombre del indicador
• fórmula
• qué mide
• interpretación
Ejemplo:
• KPI: Tasa de morosidad
• Fórmula: créditos vencidos / total de créditos
Interpretación: mide el nivel de riesgo financiero
4.6 Identificación de problemas
A partir del análisis, el postgraduante deberá identificar problemas como:
• bajo rendimiento
• alta morosidad
• ineficiencia operativa
• patrones negativos en los datos
También puede identificar:
• problemas de calidad de datos
• inconsistencias
• ausencia de indicadores

1. Plataformas generales de datos abiertos
1.1 Portal de Datos Abiertos de Bolivia
https://datos.gob.bo
Descripción:
Portal oficial del Estado boliviano que contiene datasets públicos relacionados con:
• economía
• salud
• educación
• transporte
• estadísticas nacionales
Uso recomendado:
Ideal para trabajos con enfoque local o institucional.
1.2 Data.gov (Estados Unidos)
https://data.gov
Descripción:
Uno de los repositorios más grandes de datos abiertos a nivel mundial, con miles de datasets en
múltiples áreas.
Uso recomendado:
Análisis de políticas públicas, economía, transporte, medio ambiente.
1.3 Google Dataset Search
https://datasetsearch.research.google.com
Descripción:
Motor de búsqueda especializado en datasets.
Uso recomendado:
Encontrar datasets específicos por tema (finanzas, salud, comercio, etc.).
1.4 Kaggle Datasets
https://www.kaggle.com/datasets
Descripción:
Plataforma ampliamente utilizada en ciencia de datos, con datasets organizados por temática y
nivel de complejidad.
Uso recomendado:
Ideal para estudiantes que deseen trabajar con datos estructurados y listos para análisis.
